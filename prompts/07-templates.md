# Prompt 07: Generate Document Templates

## Context

You are building a multi-role software engineering agent framework. The agent generates documents as a side effect of its work — evaluation reports, progress updates, issue comments. These documents use Jinja2 templates for consistent structure.

## Objective

Create 3 Jinja2 template files in `templates/`.

## Instructions

### `templates/confluence/evaluation-report.md.j2`

A Jinja2 template for domain evaluation reports published to the documentation platform. Must include:

1. **Header**: Title with `{{ config_name }}` vs `{{ baseline_name }}`, date, author ("Agent"), issue ID, PR URL — all as template variables with defaults
2. **Executive Summary**: `{{ executive_summary }}` variable
3. **Configuration Changes**: `{{ config_changes }}` variable
4. **Results Overview**: Table generated from `{% for metric in metrics %}` loop with columns: Metric, Baseline, New, Delta, Significant?
5. **Regression Analysis**: Conditional section (`{% if regressions %}`) with table: Query ID, Query Text, Baseline, New, Delta
6. **Improvement Analysis**: Same conditional structure as regressions
7. **Recommendations**: `{{ recommendations }}` variable
8. **Footer**: "Generated by [Agent Name]"

### `templates/confluence/progress-update.md.j2`

A Jinja2 template for periodic progress updates. Must include:

1. **Header**: Title with `{{ date_range }}`, generated date, author
2. **Completed**: Loop over `{{ completed_tasks }}` — each with title, issue ID, summary, PR URL, docs URL
3. **In Progress**: Loop over `{{ active_tasks }}` — each with title, issue ID, status, description
4. **Key Findings**: Loop over `{{ findings }}`
5. **Metrics**: Table from `{{ metrics }}` loop — name, current, previous, change
6. **Next Steps**: Loop over `{{ next_steps }}`
7. **Footer**

### `templates/jira/task-comment.md.j2`

A Jinja2 template for structured comments on issues. Must include:

1. **Header**: "Agent Update: {{ status }}"
2. **Action summary**: `{{ action_summary }}`
3. **Links**: Branch, PR URL, docs URL — each conditional (`{% if branch %}`)
4. **Details**: Conditional details section
5. **Evaluation summary**: Conditional evaluation results
6. **Next steps**: Conditional loop over next steps
7. **Footer**

## Quality Criteria

- Templates must use proper Jinja2 syntax (`{{ }}` for variables, `{% %}` for logic)
- All dynamic sections must have `{% if %}` guards for optional data
- Tables must use markdown format
- Templates should render cleanly even when optional sections are empty
- Use `| default("N/A")` for fields that might be missing
