# Phase 4: Architect + Data Scientist Roles (Weeks 10-12)

## Objective

Implement the analytical roles — the Architect who designs systems and evaluates tradeoffs, and the Data Scientist who analyzes data and extracts insights.

---

## Deliverables

### 4.1 Architect Role: System Design

- Analyze existing system architecture from code
- Propose designs with component diagrams and data flows
- Evaluate technology options with tradeoff analysis
- Generate Architecture Decision Records (ADRs)
- Identify non-functional requirements (scalability, reliability, security)

### 4.2 Architect Role: Technical Assessment

- Assess technical debt and recommend priorities
- Review system for performance bottlenecks
- Evaluate migration paths between technologies
- Design API contracts and interface specifications

### 4.3 Data Scientist Role: Data Analysis

- Connect to data warehouses (BigQuery, Snowflake, etc.)
- Run exploratory data analysis
- Execute parameterized SQL queries
- Calculate metrics with statistical rigor
- Generate insight summaries with confidence levels

### 4.4 Data Scientist Role: Experiment Design

- Design A/B tests (sample size, duration, metrics)
- Analyze experiment results with significance testing
- Build and validate ML models
- Create data visualizations for reports

### 4.5 Analysis Report Generation

- Structured reports suitable for documentation platform
- Charts and tables for visual communication
- Recommendations based on data findings
- Confidence intervals and caveats

---

## Verification Checklist

- [ ] `agent design "Notification system for 10K concurrent users"` produces architecture
- [ ] ADRs follow standard format with context, decision, consequences
- [ ] `agent analyze "Conversion funnel for Q1"` returns data-backed insights
- [ ] Statistical results include significance tests and confidence intervals
- [ ] Architecture diagrams are clear and correct
- [ ] Analysis reports include actionable recommendations
